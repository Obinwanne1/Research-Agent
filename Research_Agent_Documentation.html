Section 01

System Overview

Research Agent is a full-stack, self-hosted AI research automation platform. It converts any research topic into a structured, sourced markdown summary in under 90 seconds — using DuckDuckGo for web discovery and Claude (via CLI subprocess or SDK) for AI synthesis.

What the system does

  • Research — Topic → search → fetch → Claude → 600–1000 word summary with verified sources
  • Job Search — Role + location → DuckDuckGo → Claude JSON extraction → structured cards
  • Prompt Generator — Description → optimised AI prompt with token estimate + GPT-4 comparison
  • Skill Generator — Workflow description → Claude Code .md skill file with triggers + steps
  • Scheduled Research — Daily/weekly recurring jobs, daemon scheduler, DB-backed
  • Article Comparison — Up to 4 articles side-by-side, Claude synthesis
  • Public Sharing — Unique token link, no login required for reader
  • REST API — API key auth, rate limited, works with n8n, Zapier, scripts
  • Admin Panel — User management, job monitor, stats dashboard

Research Pipeline

Cache check
→
DDGS search
→
Cache write
→
Page fetch
→
Claude prompt
→
Source append
→
Save .md + DB

Step-by-step

1. Cache check

Key: topic.lower().strip(). If a matching row exists in search_cache and cached_at is within SEARCH_CACHE_TTL_HOURS (default 6h), the cached results list is used — no DDGS call made.

2. DuckDuckGo search

Up to 3 query variations per topic (e.g. topic, topic + "overview", topic + "latest"). DDGS().text(query, max_results=10). Results deduplicated by URL. time.sleep(1) between queries to avoid rate limiting.

3. Domain filtering

Skip results where the URL contains blocked TLDs (.cn .ru .jp .kr) or blocked patterns (baidu zhidao sina weibo sogou). Keeps English-language sources.

4. Page fetch

GET request, 10-second timeout, text/html content-type check only. HTML stripped with regex. First 8000 characters of plain text kept. Up to 5 pages fetched. Only pages returning HTTP 200 are included.

5. Claude summarise

Prompt constructed: topic + fetched page content. Sent to call_claude(). Expected output: markdown with exactly ## Overview and ## Key Findings sections. 600–1000 words. No preamble.

6. Source verification & append

Only URLs that returned HTTP 200 during fetch are appended as ## Sources. No Claude-generated or unverified URLs are ever included. Sources render as clickable links.

7. Save

Written to research/<user_id>/YYYY-MM-DD_<slug>.md with encoding='utf-8'. models.create_article() inserts DB record and FTS5 index. models.update_job(status='done', result_slug=slug). Frontend polling detects status change, redirects to article.

Section 06

Claude Integration

utils.py — single entry point

All pipeline files import call_claude or call_claude_fast from utils.py. No Claude-specific code exists anywhere else.

CLI path (_run_cli)

subprocess.run([CLAUDE_CMD, "-p"], input=prompt, capture_output=True, text=True, encoding='utf-8', timeout=timeout)

CP1252→UTF-8 re-encode attempted on output (Windows encoding artefact fix). Non-zero returncode raises RuntimeError.

SDK path (_run_sdk)

anthropic.Anthropic().messages.create(model="claude-sonnet-4-6", max_tokens=4096, messages=[...])

Only called if CLI raises an exception AND ANTHROPIC_API_KEY is set in environment. Silent fallback — pipelines never know which path ran.

Claude CMD resolution

At module load, _resolve_claude() searches:

  1. shutil.which("claude", path=npm_bin + PATH) where npm_bin = ~/AppData/Roaming/npm/
  2. Direct claude.cmd path in npm_bin if shutil.which fails
  3. Fallback to bare "claude" string (relies on system PATH)

Result stored in module-level CLAUDE_CMD. Resolved once at startup.

Timeout tiers

Section 07

Background Jobs & Scheduler

Job Queue (background.py)

ThreadPoolExecutor(max_workers=MAX_CONCURRENT_JOBS) — bounded. Prevents Claude call storms. enqueue(job_type, payload, user_id):

  1. Creates DB record in research_jobs (status: pending)
  2. Submits _run_safe(handler, payload, user_id, job_id) to executor
  3. Returns job_id immediately to caller

_run_safe catches all exceptions and sets job status to error — handlers never crash the pool.

Job Recovery on Restart

recover_pending_jobs() is called once at startup (lock-guarded):

  1. reset_stuck_jobs() — sets all running jobs back to pending (they were killed mid-flight)
  2. Fetches all pending jobs. Restores payload from DB JSON. Resubmits to executor.

Scheduler Daemon

start_scheduler() launches a single daemon thread named ra_scheduler. Loop:

while True:
    sleep(60)
    due = models.get_due_schedules()  # active WHERE next_run_at <= utcnow()
    for sched in due:
        delta = _FREQ_DELTA[sched["frequency"]]  # timedelta(days=1) or (weeks=1)
        next_run = (datetime.utcnow() + delta).isoformat()
        job_id = enqueue("research", {"topic": sched["topic"]}, sched["user_id"])
        models.update_schedule_after_run(sched["id"], next_run, job_id)
    # broad except: pass — never crash

REST API Reference

Authentication

All /api/* endpoints require one of:

  • X-API-Key: ra_<token> header
  • Authorization: Bearer ra_<token> header

Key is looked up by SHA-256 hash. On success, last_used_at is updated. User is stored in g.api_user.

Rate Limiting

In-memory sliding window per user_id. Window: 1 hour. Default limit: 20 calls. On breach: HTTP 429 + Retry-After: <seconds> header.

Section 09

Authentication & Security

User auth flow

  1. User submits login form (CSRF token validated)
  2. IP checked against login rate limiter (5 attempts → 15 min lockout)
  3. Email lookup → check_password_hash (werkzeug PBKDF2-SHA256)
  4. set_session(user) — populates Flask session with user_id, email, role, display_name, last_activity
  5. If must_change_password = 1, redirect to /change-password before dashboard

Session timeout

before_request middleware checks session["last_activity"] on every request. If elapsed > SESSION_LIFETIME_MINUTES (default 30), session is cleared and user redirected to login. Last activity updated on each request.

Section 10

Feature Reference

Public Article Sharing

  • Toggle endpoint: POST /article/<slug>/share — returns {"token": "...", "is_public": true}
  • Token generation: secrets.token_urlsafe(24) on first enable. Token is preserved in DB on subsequent enable/disable toggles (stable URLs).
  • Public route: GET /share/<token> — no auth required. Fetches article by token + is_public=1. Renders share.html (standalone, no sidebar).

Article Comparison

  • Route: GET /compare?slugs=a,b,c,d (max 4 slugs)
  • Reads each article .md file, renders to HTML via python-markdown
  • Calls Claude for synthesis (one combined prompt). If Claude fails, synthesis panel shows "unavailable" — columns still render.
  • CSS grid: grid-template-columns: repeat(N, 1fr). Each column: max-height: 70vh; overflow-y: auto.
  • h1 override in compare context: font-size: 0.95rem (prevents huge headings in narrow columns)
  • JS: checkboxes on article cards, floating compare bar (#compare-bar), navigates on click

Scheduled Research

  • Frequencies: daily = timedelta(days=1) | weekly = timedelta(weeks=1)
  • next_run_at stored as UTC ISO string. Scheduler compares against datetime.utcnow().
  • On pause: is_active = 0 — scheduler skips it. next_run_at preserved.
  • On resume: is_active = 1 — if next_run_at is in the past, fires on next 60s tick.

Search Result Cache

  • Key: topic.lower().strip() — case/whitespace normalised
  • Hit: skip DDGS call entirely, use cached results JSON, log "Using cached search results..."
  • Miss or expired: run DDGS, write cache with UPSERT (ON CONFLICT(topic_key) DO UPDATE)
  • TTL check: Python-side (datetime.utcnow() - cached_at).total_seconds() > ttl_hours * 3600

Full-Text Search

  • SQLite FTS5, tokenize='unicode61' (handles Unicode, punctuation)
  • Query: special chars stripped, words joined with spaces → MATCH ?
  • Results ordered by FTS5 rank (BM25 relevance)
  • Dashboard search: GET /dashboard?q=query — passes to search_articles(), renders filtered card grid

Section 14

Deployment Guide

Pre-deployment checklist

  • Set strong random SECRET_KEY in .env
  • Set DEBUG=false
  • Add HTTPS reverse proxy (nginx / Caddy) — app sets Strict-Transport-Security + Secure cookie when DEBUG=false
  • Never commit .env or research_agent.db
  • Regular backups of research_agent.db and research/ folder

Production server

The app uses Waitress at startup — no Gunicorn or uWSGI needed:

from waitress import serve
serve(app, host="0.0.0.0", port=Config.PORT)

Nginx reverse proxy

server {
    listen 443 ssl;
    server_name yourdomain.com;
    location / {
        proxy_pass http://127.0.0.1:5001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-Proto https;
    }
}
