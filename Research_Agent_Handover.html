Chapter 01

Platform Overview & What's New

Research Agent is a full-stack AI research automation platform. It turns any research topic into a structured, sourced summary in under 90 seconds — using DuckDuckGo for web discovery and Claude (via CLI subprocess) for AI summarisation. Version 2.0 adds REST API automation, recurring schedules, article comparison, public sharing, rate limiting, search caching, and a Claude SDK fallback.

1.1 Platform Feature Summary

AI Research

Topic → DuckDuckGo → fetch → Claude → 600–1000 word markdown summary with sources

Job Search

Role + location → structured cards with title, company, salary, Apply links

Prompt Generator

Creates optimised AI prompts with token estimate and GPT-4 comparison table

Skill Generator

Creates Claude Code skill .md files with triggers, steps, and constraints

Scheduled Research

Daily/weekly recurring research — daemon scheduler fires topics automatically

Article Comparison

Compare up to 4 articles side-by-side. Claude synthesises a combined summary

Public Sharing

Share any article via unique link — no login required for reader

REST API

API key auth — automate research from scripts, n8n, Zapier, cron jobs

Full-Text Search

SQLite FTS5 search across all article titles and topics instantly

Admin Panel

User management, role control, job monitor, stats dashboard

Dark Mode

One-click dark theme, persisted in localStorage across sessions

Markdown Export

Download any article as a raw .md file for offline use or sharing

Setup & Running the App

The app runs locally on Windows with Python and Claude Code CLI. No cloud deployment or API key is required for standard operation. An optional Anthropic API key enables the SDK fallback path.

2.1 Requirements

pipAnyPython package installer

2.2 Install & Start

  1. Install Claude Code CLI: npm install -g @anthropic-ai/claude-code then run claude to authenticate.
  2. Navigate to the project folder in PowerShell.
  3. Install Python packages: pip install -r requirements.txt
  4. Start the server: python app.py
  5. Open browser at http://localhost:5001
  6. Register your first account — it automatically becomes superadmin.
Note If port 5001 is in use: netstat -ano | findstr :5001 → get PID → taskkill /PID <pid> /F

2.3 Environment Configuration

Create a .env file in the project root:

SECRET_KEY=change-this-to-something-random-in-production
PORT=5001
DEBUG=false
MAX_CONCURRENT_JOBS=10
CLAUDE_TIMEOUT=120
CLAUDE_FAST_TIMEOUT=60
SESSION_LIFETIME_MINUTES=30
API_RATE_LIMIT=20
SEARCH_CACHE_TTL_HOURS=6
# Optional — enables Claude SDK fallback when CLI is unavailable:
ANTHROPIC_API_KEY=sk-ant-...
Important Change SECRET_KEY before any deployment. The default value is insecure.
Chapter 03

Core Features — User Guide

All features are accessible from the dashboard sidebar. Research, schedules, and the API all flow through the same job queue — status is tracked live via polling.

3.1 Research Pipeline

Submit a topic from the dashboard. The pipeline runs in the background:

  1. Cache check — if this topic was searched within the last 6 hours, results are reused from SQLite (no DuckDuckGo call).
  2. Web search — DuckDuckGo fetches top 10 results per query (up to 3 query variations). No API key required.
  3. Page fetch — top 5 English-language pages downloaded (10s timeout), HTML stripped, first 8000 characters kept. Blocked domains (.cn, .ru, .jp, baidu etc.) are skipped.
  4. Claude summarise — structured prompt sent to claude -p subprocess. Output: 600–1000 word summary in ## Overview / ## Key Findings format.
  5. SDK fallback — if CLI fails, retries via anthropic Python SDK (only if ANTHROPIC_API_KEY is set).
  6. Save — appends ## Sources from confirmed HTTP-200 URLs (no hallucinated links). Saves to research/<user_id>/YYYY-MM-DD_<slug>.md. Indexed in FTS5.

Frontend polls /api/status/<job_id> every 1.5 seconds. Spinner disappears when done, article appears in library.

3.2 Job Search

Enter a role and location (e.g. "Python developer remote"). The pipeline searches DuckDuckGo for job listings and sends results to Claude, which extracts structured JSON: [{title, company, location, salary, url, tags}]. Results render as styled cards with Apply buttons.

3.3 Prompt & Skill Generators

Prompt Generator

Describe what you need. Claude returns an optimised prompt with token estimate and a comparison table showing how Claude vs GPT-4 handles each parameter.

Skill Generator

Describe a workflow. Claude returns a complete Claude Code .md skill file with YAML frontmatter (triggers, steps, constraints, edge cases).

Both outputs are saved to the article library and searchable via FTS5.

3.4 Scheduled Research

Access via Sidebar → Schedules.

  • Create a schedule: topic + frequency (daily or weekly)
  • Schedules store next_run_at (UTC) in the database
  • A daemon thread wakes every 60 seconds, queries for due schedules (next_run_at <= now), fires them as research jobs, and advances next_run_at by the frequency delta
  • Pause/resume individual schedules without deleting them
  • Last run time and last job ID shown in the table
Note Schedules only fire while the server is running. If the server restarts, the next check will catch any overdue schedules immediately.

3.5 Article Comparison

  1. On the dashboard, tick checkboxes on article cards (up to 4).
  2. A floating compare bar appears at the bottom of the screen.
  3. Click Compare Articles — navigates to /compare?slugs=a,b,c,d.
  4. Articles render in a CSS grid, each column independently scrollable (max 70vh).
  5. Claude generates a synthesis panel below the columns (requires CLI/SDK to be reachable; silently omitted if unavailable).

3.6 Public Article Sharing

  1. Open any article.
  2. Click Make Public — generates a unique share token (secrets.token_urlsafe(24)).
  3. A Copy Link button appears showing the /share/<token> URL.
  4. Anyone with that link can read the article — no account required.
  5. Click Make Private to revoke access. Token is preserved so re-enabling restores the same link.

3.7 REST API & API Keys

Generate a key: Sidebar → API Keys → enter a name → Generate Key. Copy immediately — the raw key is shown once only (stored as SHA-256 hash).

EndpointMethodPurpose
/api/researchPOSTStart a research job. Body: {"topic": "..."}
/api/jobs/searchPOSTStart a job search. Body: {"query": "..."}
/api/status/<job_id>GETPoll job status. Returns status, message, result_slug.

All API endpoints accept authentication via:

  • X-API-Key: ra_... header
  • Authorization: Bearer ra_... header

Rate limit: 20 calls per user per hour (sliding window, in-memory). Exceeding returns HTTP 429 with a Retry-After header indicating seconds to wait.

curl -X POST http://localhost:5001/api/research \
  -H "X-API-Key: ra_your_key_here" \
  -H "Content-Type: application/json" \
  -d '{"topic": "quantum computing 2025"}'

# Response: {"job_id": 42, "status": "pending"}

curl -H "X-API-Key: ra_your_key_here" \
  http://localhost:5001/api/status/42

# Response: {"status": "done", "result_slug": "2026-06-25_quantum-computing"}

Architecture & Technical Reference

The app is a monolithic Flask application backed by a single SQLite file. Jobs run in a bounded ThreadPoolExecutor. A daemon scheduler thread handles recurring research. All Claude calls go through a single utility function with CLI-first, SDK-fallback design.

4.3 Claude Integration

Primary path: subprocess.run([CLAUDE_CMD, "-p"], input=prompt, ...)

CLAUDE_CMD is resolved at startup by searching the npm global bin directory (~/AppData/Roaming/npm/) before falling back to the system PATH. This handles the common Windows issue where claude.cmd is not on the default PATH.

Fallback path: If the CLI subprocess raises any exception AND ANTHROPIC_API_KEY is set in the environment, the call is retried via the anthropic Python SDK using claude-sonnet-4-6.

Two timeout tiers: call_claude() uses 120s (research summarisation); call_claude_fast() uses 60s (prompt/skill generation).

4.4 Background Jobs & Scheduler

Job Queue

ThreadPoolExecutor(max_workers=10) — bounded concurrency. enqueue() creates a DB record then submits to the pool. On restart: stuck jobs reset to pending and resubmitted from their stored payload.

Scheduler

Daemon thread sleeps 60s, queries schedules WHERE next_run_at <= now AND is_active = 1, fires each as a research job, and advances next_run_at. Exceptions are swallowed so the thread never crashes.

The admin panel is accessible to users with role admin or superadmin. Superadmins can promote/demote other users. Admins can view and manage all users and jobs.

Chapter 07

Deployment & Production Notes

The app is production-ready for personal and small-team use as-is. For public-facing deployment, apply the following configuration and infrastructure changes.

Pre-Deployment Checklist

  • Set a strong random SECRET_KEY in .env
  • Set DEBUG=false in .env
  • Add HTTPS via a reverse proxy (nginx or Caddy) — the app will then set Strict-Transport-Security and Secure cookie flag automatically
  • Never commit research_agent.db or .env to version control
  • Back up research_agent.db and the research/ folder regularly
  • Consider increasing MAX_CONCURRENT_JOBS for multi-user deployments
  • Set API_RATE_LIMIT appropriate to your user base

Running with Waitress (Production Server)

The app already uses Waitress when launched via python app.py:

from waitress import serve
serve(app, host="0.0.0.0", port=Config.PORT)

Waitress is a production-grade WSGI server — no Gunicorn or uWSGI needed for this scale.

Nginx Reverse Proxy (Example)

server {
    listen 443 ssl;
    server_name yourdomain.com;

    location / {
        proxy_pass http://127.0.0.1:5001;
        proxy_set_header Host $host;
